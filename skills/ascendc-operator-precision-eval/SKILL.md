# AscendC 算子精度验证 Skill

## 功能概述
该 Skill 基于 `ct_project` 工具，提供 AscendC 算子精度验证的完整解决方案，包括精度概念介绍、测试报告生成、问题分析和定位策略等功能。

## 核心功能模块

### 1. 精度概念介绍

#### 1.1 单标杆验证
- **定义**：与单一参考平台（通常为 CPU）进行精度比对
- **核心逻辑**：待测数据与 CPU 高精度实现结果进行逐点比较
- **适用场景**：快速验证、已有成熟参考实现的算子

#### 1.2 双标杆验证
- **定义**：同时与真值（Golden）和标杆（Bench）进行精度比对
- **核心逻辑**：计算 (待测vs真值) 与 (标杆vs真值) 的各项误差比值
- **适用场景**：新算子开发、跨平台迁移验证、高精度要求场景

#### 1.3 精度标准
- **动态阈值**：根据数据类型和计算次数自动调整误差阈值
- **小值域处理**：对接近零的值采用特殊的绝对误差判定
- **Inf/NaN 检查**：专门检查特殊值的一致性

#### 1.4 核心问题解答

**算子精度标准是什么？**
```
# 单标杆判定公式
|actual - golden| <= err * max(Small_Val_Thd, |golden|)

# 双标杆判定逻辑
计算待测数据与真值的误差指标 / 标杆数据与真值的误差指标
若该比值 <= 预设阈值，则通过
```

**算子精度标准怎么用？**
```bash
# 单标杆测试
ct single test_data.npy golden_data.npy --calc_count 4096

# 双标杆测试
ct dual test_data.npy golden_data.npy bench_data.npy --level L1
```

**算子精度评价维度（哪些统计指标）？**
- **MARE**：最大相对误差
- **MERE**：平均相对误差  
- **RMSE**：均方根误差
- **ERR_COUNT**：小值域错误点数

**算子精度报告怎么看？**
- 先看总体结果（PASS/FAIL）
- 检查各项误差指标是否在阈值范围内
- 分析误差分布和异常值情况
- 关注诊断信息中的失败原因

```bash
ct dual ... --level L2 --MARE_ratio_thd 1.5 --MERE_ratio_thd 1.2
```

### 3. 精度问题分析

#### 3.1 精度误差
- **定义**：误差在可接受范围内的偏差
- **常见原因**：
  - 浮点运算精度限制
  - 计算顺序差异
  - 编译器优化影响

#### 3.2 精度错误
- **定义**：误差超出可接受范围的问题
- **常见原因**：
  - 算法实现错误
  - 数据类型不匹配
  - 内存访问错误
  - 并行计算同步问题

#### 3.3 核心问题解答

**为什么精度没有通过？**
1. **检查数据类型**：确保测试数据与参考数据类型一致
2. **检查计算次数**：对于累积计算，需要正确设置 `--calc_count`
3. **分析误差分布**：使用 `ct viz` 查看误差分布情况
4. **查看诊断信息**：CT工具会提供详细的失败诊断信息
5. **检查边界条件**：特殊输入值可能导致精度问题

### 4. 精度问题定位策略

#### 4.1 数值聚类分析
- **方法**：分析误差值的分布和统计特征
- **工具**：CT工具内置的分布分析功能
- **应用**：
  ```bash
  ct dual test.npy golden.npy bench.npy --save_chart
  ```

#### 4.2 位置聚类分析
- **方法**：分析误差在张量中的空间分布
- **工具**：`ct viz --spatial` 生成空间热力图
- **应用**：
  ```bash
  ct viz test.npy golden.npy --spatial --out_dir ./spatial_analysis
  ```

#### 4.3 核心问题解答

**怎么解决精度问题？**

1. **算法层面**：
   - 检查算法实现与参考版本的一致性
   - 优化数值稳定性
   - 调整计算顺序减少累积误差

2. **实现层面**：
   - 确保数据类型匹配
   - 修复内存访问错误
   - 解决并行计算同步问题

3. **工具辅助**：
   - 使用 `ct single`/`ct dual` 进行快速验证
   - 使用 `ct viz` 进行可视化分析
   - 根据CT工具的诊断信息定位问题

## CT工具核心功能详解

### 1. 单标杆验证（single）

#### 核心配置
- **计算次数（calc_count）**：决定误差阈值的关键参数
  - 对于Element-wise操作：1
  - 对于矩阵乘法：K轴大小
  - 对于累积操作：累积次数

- **数据类型（dtype）**：支持 float32、float16、bfloat16 等

- **小值域阈值**：根据数据类型自动设定
  - float32: 2^-14
  - float16: 2^-11
  - bfloat16: 2^-8

#### 执行流程
1. 数据加载与预处理
2. 逐点精度检查
3. 统计误差指标
4. 生成测试报告

### 2. 双标杆验证（dual）

#### 严格等级
| Level | 描述 | MARE_ratio | MERE_ratio | RMSE_ratio | ERR_COUNT_ratio |
|-------|------|------------|------------|------------|----------------|  
| L0 | 宽松模式 | 10.0 | 2.0 | 2.0 | 2.0 |
| L1 | 标准模式 | 5.0 | 1.5 | 1.5 | 2.0 |
| L2 | 严格模式 | 2.0 | 1.2 | 1.2 | 2.0 |

#### 关键指标
- **MARE**：最大相对误差
- **MERE**：平均相对误差
- **RMSE**：均方根误差
- **ERR_COUNT**：小值域错误点数

#### 执行流程
1. 数据加载与预处理
2. Inf/NaN 特殊值检查
3. 误差指标计算
4. 比值分析与判定
5. 失败诊断（可选）
6. 生成测试报告

### 3. 可视化工具（viz）

#### 功能
- **误差分布图**：展示误差的分布情况
- **空间热力图**：展示误差在张量中的空间分布
- **失败点标记**：高亮显示误差超过阈值的点

#### 应用场景
- 快速识别误差模式
- 定位空间上集中的误差
- 直观展示精度问题

## 最佳实践

### 1. 数据准备
- **覆盖边界情况**：包含最大值、最小值、零值等
- **多样化数据分布**：均匀分布、正态分布、稀疏数据等
- **适当数据规模**：既保证测试效率，又能发现问题

### 2. 测试策略
- **从单标杆开始**：快速验证基本正确性
- **逐步增加复杂度**：从小规模输入到大规模输入
- **结合双标杆**：在关键阶段进行更严格的验证

### 3. 问题排查
- **使用可视化工具**：直观了解误差分布
- **分析失败样本**：查看CT工具提供的失败样本详情
- **对比中间结果**：将复杂算子拆分为多个步骤逐一验证
- **调整测试参数**：尝试不同的计算次数和数据类型

## 常见问题与解决方案

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 通过率低 | 计算次数设置错误 | 正确设置 `--calc_count` 参数 |
| 小值域误差大 | 小值域阈值不合适 | 调整 `--small_val_thd` 参数 |
| Inf/NaN 失败 | 特殊值处理不一致 | 检查特殊值的处理逻辑 |
| 空间集中误差 | 边界条件处理错误 | 使用 `--spatial` 查看空间分布，检查边界处理 |
| 平台差异大 | 数据类型不匹配 | 确保测试数据与参考数据类型一致 |
