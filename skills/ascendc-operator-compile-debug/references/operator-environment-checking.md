## 概述

环境检查是算子编译部署前的必要步骤，确保所有依赖都已正确配置，避免后续编译过程中出现错误。

## 环境检查流程

在开始编译前，必须进行全面的环境检查，确保所有依赖都已正确配置。可以通过执行`scripts/check_environment.sh`脚本快速完成环境检查：

```bash
# 执行环境检查脚本
bash $SKILL_DIR/scripts/check_environment.sh

# 根据返回值进行后续操作
if [ $? -eq 0 ]; then
    echo "环境检查通过，继续执行编译操作"
else
    echo "环境检查失败，请修复问题后重试"
    exit 1
fi
```

脚本将检查以下内容：

- CANN环境变量设置
- Python版本（3.7.0-3.10）
- GCC版本（≥7.3.0）
- CMake版本（≥3.16.0）
- 必要工具（g++、dos2unix、gawk、make等）
- NPU设备信息
- PYTHONPATH配置

所有检查通过后，脚本将返回0并继续执行后续操作；否则返回1并提示错误信息。

## 依赖工具检查

确保以下工具已安装并满足版本要求：

| 工具名称 | 版本要求 | 用途 |
|---------|---------|------|
| Python3 | 3.7.0-3.10 | 运行msOpGen等工具 |
| gcc | ≥7.3.0 | 编译C/C++代码 |
| g++ | 与gcc兼容 | 编译C++代码 |
| CMake | ≥3.16.0 | 构建项目 |
| dos2unix | 任意版本 | 转换文件格式 |
| gawk | 任意版本 | 文本处理 |
| make | 任意版本 | 执行构建命令 |

## CANN环境检查

确保CANN环境变量已正确设置：

- `ASCEND_HOME_PATH`：CANN安装路径
- `PYTHONPATH`：包含CANN Python路径
- 其他CANN相关环境变量

## 算子工程目录检查

在环境检查完成后，还需要检查算子工程目录结构是否完整：

```bash
# 检查算子工程目录结构是否完整
if [ ! -f "CMakeLists.txt" ]; then
    echo "错误：当前目录下未找到CMakeLists.txt文件"
    echo "请确保当前目录是正确的算子工程目录"
    exit 1
fi

if [ ! -d "op_host" ] || [ ! -d "op_kernel" ]; then
    echo "错误：算子工程目录结构不完整"
    echo "缺少op_host或op_kernel目录"
    exit 1
fi
```

## 最佳实践

1. **全面检查**：在编译前执行所有环境检查，避免后续出现问题
2. **版本匹配**：确保所有工具版本与CANN版本兼容
3. **路径正确**：所有环境变量和路径配置必须准确无误
4. **错误预判**：提前检查可能导致编译失败的因素
5. **用户友好**：提供清晰的错误信息和解决方案
